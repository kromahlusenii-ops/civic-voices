# Test: Search to Report

**Persona**: Research Analyst — Policy researcher or journalist tracking public discourse across social platforms.

**Tech level**: Intermediate | **Patience**: 7/10

---

## Pre-test

- [ ] Start screen recording (e.g., Loom, OBS, or QuickTime)
- [ ] Clear browser state (cookies, cache) or use incognito
- [ ] Have test credentials ready (email/password or Google) if needed
- [ ] Open the app at http://localhost:3000 (or your test URL)

---

## Instructions

Read each step out loud before attempting it. **Think aloud** as you work—describe what you see, what you expect, and any confusion. This helps reviewers follow along.

---

## Steps

### Step 1: Land on search

- **Goal**: Reach the main search interface to start researching.
- **Do**: Navigate to the app. Click "Try for free" or "Log in" to reach the search interface.
- **Think aloud**: Is it obvious where to start? Any distracting elements?
- **Success**: Search page is visible with a query input, source filters (X, TikTok, etc.), and time/language filters.

---

### Step 2: Enter search query

- **Goal**: Run a cross-platform search to aggregate posts.
- **Do**: Type a topic in the search box (e.g., "climate policy 2024", "electric vehicles adoption"). Optionally select platforms and time range. Click the search/arrow button.
- **Think aloud**: Is the query box easy to find? Do the filters make sense?
- **Success**: Search executes; loading state appears, then results show. Left panel shows AI interpretation and key themes. Right panel shows post cards with platform, author, and engagement.

---

### Step 3: Review AI insights

- **Goal**: Understand the AI-generated summary of the conversation.
- **Do**: Read the AI interpretation, key themes, and sentiment breakdown. Note any suggested follow-up queries. Click a suggested query if interested.
- **Think aloud**: Is the interpretation useful? Are the themes accurate? Would you trust these insights?
- **Success**: Interpretation is readable and relevant. Themes are displayed. Sentiment summary visible. Suggested queries are clickable.

---

### Step 4: Sign in to save (if not authenticated)

- **Goal**: Authenticate to save search history and generate reports.
- **Do**: If prompted to sign in to save search, complete signup or login (email/password or Google).
- **Think aloud**: Was the auth flow clear? Any friction?
- **Success**: Auth modal appears and can be completed. User returns to search results. Search is saved to history.

---

### Step 5: Generate report

- **Goal**: Create a professional report with charts and analytics.
- **Do**: Click "Start research" or "Generate report" (or equivalent CTA). Wait for report generation to complete.
- **Think aloud**: Is the CTA obvious? How long does it feel? Is progress clear?
- **Success**: Report generation starts (progress indicator). Report page loads with metrics row, activity chart, sentiment/emotions breakdown, top posts, topics table.

---

### Step 6: Explore report dashboard

- **Goal**: Explore the data and verify credibility badges.
- **Do**: Scroll through report sections. Click tabs (e.g., Content Breakdown, Topics). Sort tables if available. Click a post to open source URL.
- **Think aloud**: Are the charts understandable? Do credibility badges help? Is anything missing?
- **Success**: All sections render. Tabs switch. Sorting works. Post links open external platform. Credibility badges (Official, News, Journalist, Expert) visible where applicable.

---

### Step 7: Access search history

- **Goal**: Return to a previous search.
- **Do**: Open the search history sidebar (clock/history icon). Click a previous search to reload it.
- **Think aloud**: Is the history easy to find? Does the grouping help?
- **Success**: Sidebar shows saved searches grouped by date. Clicking a search navigates and loads that query.
